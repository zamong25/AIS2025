# 델파이 트레이더 - 우리가 원하는 것

## 🎯 핵심 목표

### 1차 목표: 자동화된 수익 창출
- **24시간 자동 거래**: 사람의 개입 없이 지속적으로 작동
- **안정적 수익**: 월 평균 5-10% 수익률 목표
- **리스크 제한**: 최대 손실 월 10% 이내

### 2차 목표: 지속 가능한 시스템
- **자가 학습**: 거래 결과를 분석하여 전략 개선
- **장애 복구**: 오류 발생 시 자동 복구
- **확장 가능**: 새로운 전략/거래소 쉽게 추가

## 💡 핵심 아이디어

### "5개의 전문가가 토론하여 결정"
1. **차티스트 (Chartist)**: 차트 패턴 분석
2. **저널리스트 (Journalist)**: 뉴스와 시장 심리 분석
3. **퀀트 (Quant)**: 통계적/수학적 분석
4. **스토익 (Stoic)**: 리스크 관리 전문가
5. **신디사이저 (Synthesizer)**: 최종 의사결정자

### "트리거 기반 대응"
- 지속적 모니터링 대신 이벤트 기반 작동
- 중요한 가격/시간/조건에서만 분석 실행
- 리소스 효율적이고 정확한 타이밍

## 📊 성공 지표

### 정량적 지표
- **승률**: 55% 이상
- **손익비**: 1.5:1 이상
- **최대 낙폭(MDD)**: 15% 이내
- **샤프 비율**: 1.5 이상

### 정성적 지표
- **시스템 가동률**: 99% 이상
- **수동 개입 빈도**: 주 1회 이하
- **새 기능 추가 시간**: 1일 이내

## 🛡️ 핵심 원칙

### 1. 보수적 접근
- 확실하지 않으면 거래하지 않음
- 작은 수익을 꾸준히 추구
- 큰 손실 절대 방지

### 2. 투명성
- 모든 결정 과정 기록
- 거래 근거 명확히 설명
- 성과 정기적으로 보고

### 3. 지속성
- 단기 성과보다 장기 안정성
- 시장 변화에 적응 가능
- 기술 부채 최소화

## 🎨 사용자 경험

### 운영자 관점
- **설정 후 잊기**: 최소한의 관리만 필요
- **명확한 알림**: 중요한 이벤트만 통지
- **쉬운 모니터링**: 한눈에 보는 대시보드

### 개발자 관점
- **깨끗한 코드**: 이해하고 수정하기 쉬움
- **모듈식 구조**: 기능 추가/제거 용이
- **충분한 문서**: 의도와 구현 명확히 설명

## 🚀 장기 비전

### Phase 1 (현재-3개월)
- 단일 거래소, 단일 자산 완벽 구현
- 안정적 수익 모델 확립
- 기술 부채 제거

### Phase 2 (3-6개월)
- 멀티 자산 지원
- 고급 전략 추가
- 성능 최적화

### Phase 3 (6개월 이후)
- 멀티 거래소 지원
- 포트폴리오 관리
- AI 고도화

## ❌ 원하지 않는 것

### 기술적 측면
- 복잡한 스파게티 코드
- 하드코딩된 값
- 테스트 없는 코드
- 문서 없는 시스템

### 비즈니스 측면
- 도박성 거래
- 과도한 레버리지
- 검증 없는 전략
- 불투명한 의사결정
